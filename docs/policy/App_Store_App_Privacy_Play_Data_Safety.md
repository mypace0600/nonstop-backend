## 1️⃣ App Store **App Privacy** (iOS)

### 이게 뭐냐면

애플이 요구하는 **개인정보 영양성분표** 같은 거야 🍎
앱스토어 상세 페이지 맨 아래에 **Privacy** 섹션으로 노출돼.

> 💡 코드에서 뭐 수집하느냐 ❌
> 👉 **“우리는 이런 데이터를 이렇게 씁니다”라고 개발자가 선언** ⭕

---

### 🔍 체크하게 되는 항목들

#### ① 수집하는 데이터 유형

* 연락처 정보
* 위치
* 사용자 콘텐츠 (게시글, 사진)
* 식별자 (User ID, Device ID)
* 사용 데이터 (로그, 행동 데이터)
* 건강 / 금융 정보 등

#### ② 데이터 사용 목적

* 앱 기능 제공
* 분석
* 마케팅
* 개인화
* 보안 / 사기 방지

#### ③ 사용자와 연결되는지 여부

* 실명, 계정과 직접 연결됨?
* 익명 처리됨?

#### ④ 추적(Tracking) 여부

* 다른 앱/웹사이트 활동 추적?
* 광고 목적?

---

### ⚠️ iOS에서 진짜 중요한 포인트

* **여기 적은 내용 ≠ 실제 앱 동작**

    * 다르면? → **즉시 리젝**
* Firebase 로그 하나만 써도
  → “Usage Data 수집” 체크해야 하는 경우 많음
* “우리는 수집 안 합니다” 체크했는데

    * 회원가입 있음 → ❌

📌 **애플은 “거짓말”을 제일 싫어함**

## 2️⃣ Google Play **Data Safety** (Android)

### 이게 뭐냐면

구글판 App Privacy.
플레이스토어 상세 페이지에 **Data safety**로 표시됨 🤖

---

### 🔍 체크 항목 구조

#### ① 수집 데이터

* 개인 정보
* 앱 활동
* 위치
* 메시지
* 사진/파일

#### ② 데이터 처리 방식

* 수집 여부
* 공유 여부 (제3자)
* 암호화 여부
* 삭제 요청 가능 여부

#### ③ 보안 조치

* 전송 중 암호화
* 사용자 데이터 삭제 제공 여부

---

### ⚠️ Android에서 중요한 포인트

* **Manifest에 선언된 권한 기준으로 의심**

    * 실제로 안 써도 선언돼 있으면 물어봄
* “공유 안 함” 체크했는데

    * 광고 SDK 있음 → ❌
* 테스트 계정으로 **실제 동작 검증**함

---

## 3️⃣ 둘의 차이 한 방에 정리

| 구분    | App Store (iOS) | Play Store (Android) |
| ----- | --------------- | -------------------- |
| 관점    | 의도 + 선언         | 실제 동작                |
| 민감도   | 매우 높음           | 높음                   |
| 추적    | ATT 엄격          | 비교적 느슨               |
| 거짓 기재 | 즉시 리젝           | 수정 요구                |

---

## 4️⃣ 왜 이게 그렇게 중요하냐면

### ❌ 흔한 리젝 시나리오

* App Privacy: “수집 안 함”
* 실제 앱: 회원가입 + 로그 수집
  → **Guideline 5.1.1 위반**

### ❌ 또 다른 케이스

* Data Safety: “제3자 공유 없음”
* Firebase Analytics 사용
  → **Misleading disclosure**

---

## 5️⃣ 실무에서 이렇게 접근해라 (이게 핵심)

### 1단계

👉 **기능 기준으로 수집 데이터 목록화**

### 2단계

👉 **사용 중인 SDK 정리**

* Firebase
* Ad SDK
* Crashlytics
* Analytics

### 3단계

👉 App Privacy / Data Safety 작성

### 4단계

👉 Privacy Policy 문서와 **100% 일치**시키기
